## 📖 技能描述

本技能为 AI Agent 提供透明的上下文变量获取能力。浏览器插件会在用户访问目标网页时，自动将页面中的关键值（如 LocalStorage、SessionStorage、Cookie 中的内容）推送到本地服务。Agent 需要调用受保护的 API 时，通过本技能一条命令即可拿到对应的值，无需用户手动拷贝。

**零配置**：无需编辑任何配置文件，无需手动启动服务。脚本会自动检测并按需在后台拉起本地服务。

---

## 🚀 触发时机

1. Agent 准备请求受保护的平台 API 之前。
2. 接口请求返回 `401` 或 `403`。
3. 用户要求 Agent "调个接口"、"去平台查一下"等操作时。

---

## 🛠️ 使用方法

### 获取值

在项目根目录下执行：

```bash
node script/get_token.js <key>
```

- `key`：浏览器插件推送时使用的键名（如 `clouddevops_token`）。
- **成功**：在 `stdout` 输出原始值，状态码 `0`。
- **失败**：在 `stderr` 输出原因，状态码 `1`。

**脚本会自动处理以下情况**：
- 本地服务未运行 → 自动后台启动，无需手动干预。
- 本地服务已运行 → 直接请求，毫秒级返回。

### 列出所有已推送的值

```bash
curl http://127.0.0.1:4099/v1/credentials
```

## 🔍 决策流程（Agent 遵循）

```
Agent 需要某个平台的上下文值
 │
 ▼
执行 node script/get_token.js <key>
 │
 ├── 成功 ──▶ ✅ 拿到值，继续任务
 │
 └── 失败（key 未找到）
      │
      ▼
     用户是否已安装浏览器插件？
      │
      ├── 未安装 ──▶ 引导用户安装（见下方）
      │
      └── 已安装 ──▶ 提示用户打开/刷新目标网页，然后重试
```

## 🌐 引导用户安装浏览器插件

当 `get_token.js` 返回"未找到"时，Agent 应向用户发送以下提示（将 `<project_root>` 替换为实际绝对路径）：

> 需要先安装浏览器插件来自动同步上下文值：
> 1. 打开 Chrome，访问 `chrome://extensions/`
> 2. 开启右上角 **"开发者模式"**
> 3. 点击 **"加载已解压的扩展程序"**，选择：`<project_root>/chrome-extension`
> 4. 进入插件选项页，确认本地地址为 `http://127.0.0.1:4099`，添加抓取规则
> 5. 打开或刷新目标网页，值会自动推送到本地
>
> 完成后重新执行 `node script/get_token.js <key>` 即可。

## 💡 补充说明

- 本地服务监听端口可在 `server/config.json` 中修改（默认 `4099`）。
- 服务进程为后台常驻，多个 Agent / IDE 窗口共享同一个服务实例，互不冲突。
- 如需手动启动或重启服务：`node server/dist/index.cjs`
